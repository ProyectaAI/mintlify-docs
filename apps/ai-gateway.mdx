The Proyecta AI Gateway lets the apps you build call frontier LLMs without creating accounts with each provider, managing their API keys, or fronting the cost yourself. It speaks the OpenAI Chat Completions protocol — point the OpenAI SDK at the gateway and you're ready to build.

## Quickstart

The gateway speaks the OpenAI Chat Completions protocol. Use the official OpenAI SDK with two changes — `baseURL` and the env-var name for the key:

```ts
import OpenAI from 'openai';

const openai = new OpenAI({
  baseURL: 'https://cloud.proyecta.dev/v1/ai',
  apiKey: process.env.PROYECTA_API_KEY,
});

const response = await openai.chat.completions.create({
  model: 'openai/gpt-4o-mini',
  messages: [{ role: 'user', content: 'Summarize this article in 3 bullet points: …' }],
});
```

<Tip>
  Ask the AI builder to wire this up for you: `"Add a chatbot to my app using the Proyecta AI Gateway. The user types a question and gets a streaming response."`
</Tip>

## Streaming

Streaming works like any OpenAI-compatible client:

```ts
const stream = await openai.chat.completions.create({
  model: 'openai/gpt-4o-mini',
  messages: [{ role: 'user', content: prompt }],
  stream: true,
});

for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content ?? '');
}
```

## API keys, rate limits, and budgets

Proyecta automatically provisions a `PROYECTA_API_KEY` for your runtime. You don't need to create one manually for the common case.

For more advanced setups — multi-app organizations, model allowlists, per-key rate limits, or expiration policies — API keys can be created and managed at the organization level. Each key can carry:

- A **name** (for tracking)
- An optional **runtime scope** (lock the key to one app)
- An **allowed model list** (e.g. only Claude models)
- A **rate limit per minute**
- An **expiration**

Organization owners can also view **AI credits usage** for the current period — current spend, budget cap, and overage markup — visible in the builder's credits indicator.

## Billing

AI Gateway usage is tracked separately from the credits the AI builder consumes when it edits your app. Per-request cost is the underlying provider's rate, applied to your organization's AI credits balance.

## FAQ

<AccordionGroup>
  <Accordion title="What's the difference between the AI Gateway and the AI builder?">
    The **AI builder** is the agent that writes code in the chat panel. The **AI Gateway** is what your published app calls when *its* users want an AI feature — they are separate systems with separate billing.
  </Accordion>
  <Accordion title="Can I bring my own provider keys?">
    Yes. Ask the AI to remove the gateway calls and configure a direct integration with your own key in Environment Variables. You'll lose unified billing and the shared rate-limit pool.
  </Accordion>
  <Accordion title="Can I use it for image or voice generation?">
    Not yet. Today the gateway is text-in / text-out. Image, voice, and video modalities are on the roadmap. For image generation during the build process, see [Image Generation](/apps/image-generation).
  </Accordion>
  <Accordion title="Where can I see usage and manage keys?">
    API key management and credits usage live at the **organization** level in the Settings page. A dedicated AI Gateway panel inside the builder — with per-app keys and live usage charts — is on the roadmap.
  </Accordion>
</AccordionGroup>
