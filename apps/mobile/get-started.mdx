Proyecta builds web apps, and every web app you create is already mobile-accessible in any phone browser. From there, you have two paths: add Progressive Web App (PWA) support so users can install your app to their home screen, or wrap your published PWA with a tool like [PWA Builder](https://pwabuilder.com) to distribute it through the App Store, Google Play, or Microsoft Store.

## What you get out of the box

Every Proyecta app includes a responsive viewport switcher in the builder so you can preview your layout at Desktop, Tablet, and Mobile sizes while editing. The AI is also trained to produce responsive layouts when you prompt for them, and Proyecta's own builder UI works on phones so you can keep building on the go.

## The PWA path

For most apps, a PWA is the right answer. Ask the AI to set it up:

```text
Add PWA support to my app:
- Web app manifest with name, icons, theme color
- Service worker with offline cache for the app shell
- Install prompt that appears after the user has visited 3 times
```

The AI will configure the manifest, service worker, and install prompt. Once you publish, users can:

- Open your app in any mobile browser
- Tap the browser menu and choose **Add to Home Screen** (iOS) or **Install app** (Android)
- Launch it like a native app from their home screen

PWAs on iOS 16.4+ also support Web Push — see [Push Notifications](/apps/push-notifications) for setup.

## Going further: native store distribution

If you need to ship to the App Store or Google Play, wrap your published PWA using PWA Builder:

<Steps>
  <Step title="Publish your app">
    Publish to your `*.proyecta.dev` subdomain or a custom domain so your app has a live HTTPS URL.
  </Step>
  <Step title="Open PWA Builder">
    Go to [pwabuilder.com](https://pwabuilder.com) and enter your published URL.
  </Step>
  <Step title="Generate native packages">
    PWA Builder audits your manifest and service worker, then generates:
    - **Android** — an `.aab` bundle for Google Play
    - **iOS** — an Xcode project for the App Store
    - **Windows** — an `.msix` for the Microsoft Store
  </Step>
  <Step title="Submit to the store">
    Follow the submission process for each store. See [Publish to App Stores](/apps/mobile/publish-app-store) for store-specific details.
  </Step>
</Steps>

## Limits to know

<Warning>
  Apple may reject pure WebView apps that don't add native functionality. Strengthen your case by using web APIs for push notifications, sharing, or camera access before submitting to the App Store.
</Warning>

- **Push notifications on iOS** require the user to install the PWA from Safari first — not from a wrapper.
- **Camera, GPS, and contacts** all work via standard web APIs. Native-only APIs such as HealthKit and ARKit are not reachable from a PWA wrapper.
