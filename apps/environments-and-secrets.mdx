Secrets and environment variables let you store sensitive values — API keys, database credentials, third-party service tokens — separately from your code. They are injected into your app's runtime at startup and never committed to your repository.

## Add a secret

<Steps>
  <Step title="Open Settings">
    In the builder, navigate to the **Settings** page.
  </Step>
  <Step title="Go to Environment Variables">
    Select the **Environment Variables** section.
  </Step>
  <Step title="Enter a name and value">
    Type a name following the convention `ALL_CAPS_WITH_UNDERSCORES` (for example, `STRIPE_SECRET_KEY`) and paste in the value.
  </Step>
  <Step title="Click Add">
    The secret is saved and immediately available to your runtime.
  </Step>
</Steps>

## Use secrets in your code

Reference any secret in your app's backend code using `process.env`:

```javascript
const stripe = new Stripe(process.env.STRIPE_SECRET_KEY);
```

The AI uses your secrets automatically when you ask it to build integrations:

| Prompt | Secret used |
|---|---|
| `"Accept payments with Stripe"` | `STRIPE_SECRET_KEY` |
| `"Send emails with Resend"` | `RESEND_API_KEY` |
| `"Add Google OAuth"` | `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` |

## View and update secrets

Secret values are masked by default. You can:

- **Toggle visibility** to reveal a value
- **Update** the value in place
- **Delete** a secret and recreate it with a new value

## What the AI can see

The AI can see your **secret names** but not their values. This lets it reference the correct environment variable when building features without exposing sensitive data.

## Sensitive key detection

Proyecta automatically detects common sensitive key patterns (API keys, tokens, passwords) and masks their values in the UI — even if you forget to mark them as sensitive yourself.

## FAQ

<AccordionGroup>
  <Accordion title="Are secrets encrypted?">
    Yes. Secrets are stored securely and injected into your runtime environment at startup. They never appear in your committed code or version history.
  </Accordion>
  <Accordion title="Can I have different secrets for development and production?">
    Not currently. Environment separation is planned for a future release. Today, all secrets apply to your development runtime.
  </Accordion>
  <Accordion title="What should I do if I accidentally expose a secret?">
    Immediately delete the secret in Proyecta, revoke the key with the third-party provider, and create a new one. Never commit secrets to your code — use environment variables instead.
  </Accordion>
</AccordionGroup>

<Note>
If your app uses [Proyecta Auth](/apps/users-auth) and you add OAuth providers like Google, the AI expects credentials to follow the standard naming convention (`GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`). Using these names ensures the AI wires up the integration correctly.
</Note>
